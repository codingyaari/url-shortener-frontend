# Screenshots Guide

## Quick Steps to Add Screenshots

1. **Start your development server:**
   ```bash
   npm run dev
   ```

2. **Take screenshots** using these keyboard shortcuts:
   - **Mac**: `Cmd + Shift + 4` (select area) or `Cmd + Shift + 3` (full screen)
   - **Windows**: `Windows + Shift + S`
   - **Linux**: Use screenshot tool of your choice

3. **Save screenshots** in this folder (`docs/screenshots/`) with these exact names:
   - ✅ `dashboard.png` - Screenshot of your dashboard
   - ✅ `analytics.png` - Screenshot of analytics panel
   - ✅ `dark-mode.png` - Screenshot showing dark mode
   - ✅ `home-page.png` - Screenshot of the home/landing page

4. **Recommended image specifications:**
   - Format: PNG or JPG
   - Size: 1200-1600px width (for best GitHub display)
   - Try to capture the full viewport or key sections

5. **Optimize images** (optional):
   - Use [TinyPNG.com](https://tinypng.com/) to compress
   - Or [Squoosh.app](https://squoosh.app/) for more control
   - Keep file sizes under 500KB each

## What to Capture

### Dashboard (`dashboard.png`)
- Full dashboard view with links list
- Overview stats visible
- Clean, professional look

### Analytics (`analytics.png`)
- Analytics panel showing charts/graphs
- Click data, geographic info
- Real-time metrics

### Dark Mode (`dark-mode.png`)
- Same dashboard but in dark theme
- Shows the neon/glow effects
- Highlights the dark mode aesthetic

### Home Page (`home-page.png`)
- Landing page with hero section
- Feature highlights
- Call-to-action buttons

## Troubleshooting

**Images not showing in README?**
- Check file names match exactly (case-sensitive)
- Ensure images are in `docs/screenshots/` folder
- Verify you've committed and pushed the images to git

**Images too large?**
- Use image compression tools mentioned above
- Consider using WebP format for smaller sizes
